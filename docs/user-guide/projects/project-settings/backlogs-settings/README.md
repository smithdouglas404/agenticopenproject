# Backlogs settings

In OpenProject, you can configure your backlogs settings specific to each project.

## Set the definition of done

You can define what "done" means for the Backlogs module. This determines when a work package should be considered complete and included in the backlog views and reporting. 

Choose the status or statuses which should be treated as "done".



![Manage backlogs settings under project settings in OpenProject](openproject_user_guide_project_settings_backlogs_definition_of_done.png)

Press the **Save** button to apply your changes.

![Manage backlogs settings under project settings in OpenProject](openproject_user_guide_project_settings_backlogs_done_status.png)

## Sharing sprints

Sharing is a project-level setting that allows you choose whether sprints should be shared across projects or not.

> !(Note)
>
> This is not a sprint-level setting which is typical with versions at the moment.



**Don't share:** This is the default setting for projects. Sprints can be created in this project and will only be available and visible to this project. None of the created sprints are shared with any other project or sub-projects.

**Share sprints:** Sprints can be created in this project and shared with either **all projects** or **subprojects**

**All projects:** Selecting this option means the sprints created are available to all projects within the instance you are working in. It also means other projects will not be able to use this option.

**Subprojects:** Sprints created in this project will be available to all sub-projects of the current project.

**Receive shared sprints:** No sprints can be created within this project. Rather, only sprints shared by other projects can be used.
